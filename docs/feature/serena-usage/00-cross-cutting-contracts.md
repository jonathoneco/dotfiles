## Policy Rules

- Auto-generated `.serena/project.yml` files are project-local and should NOT be committed to git — add `.serena/` to `.gitignore` if not already present
- When multiple languages are detected in a project, list ALL detected languages in `project.yml`
- Existing `.serena/project.yml` files are never overwritten — user config takes precedence
- Serena version is NOT pinned — use latest from `git+https://github.com/oraios/serena`
- All shell scripts must pass `shellcheck -x`
- Skill files follow the existing YAML frontmatter + markdown pattern from `fix-issue/SKILL.md`

## Hook Output Format

The `serena-ensure` script outputs structured text to stdout on success. This text becomes Claude Code session context via the SessionStart hook.

```
Serena MCP server active for <project-name> (languages: <lang1>, <lang2>).
Run /serena-activate to initialize semantic code navigation tools.
```

On failure (server won't start), output nothing — the `|| true` in the hook config ensures Claude Code still starts. The skill gracefully handles missing Serena.

## Skill MCP Tool Call Sequence

The `/serena-activate` skill instructs Claude to call these MCP tools in order:

```
1. mcp__serena__check_onboarding_performed
   → Returns: "onboarding performed" or "onboarding not performed"
   → If not performed: call mcp__serena__onboarding, then follow its instructions

2. mcp__serena__initial_instructions
   → Returns: Serena's system prompt (tool usage guidance)
   → Claude reads and follows these instructions for the rest of the session
```

`mcp__serena__activate_project` is NOT called — it's disabled in `claude-code` context (`single_project: true` auto-activates).

## Project.yml Template

Auto-generated by `serena-ensure` when `.serena/project.yml` is missing:

```yaml
project_name: "<directory-name>"
languages:
  - <detected-language>
encoding: utf-8
use_gitignore: true
read_only: false
excluded_tools:
  - list_dir
  - find_file
  - search_for_pattern
  - think_about_collected_information
  - think_about_task_adherence
  - think_about_whether_you_are_done
  - summarize_changes
  - get_current_config
```

## Language Detection Map

| Marker | Language Value |
|--------|---------------|
| `go.mod` | `go` |
| `package.json` | `typescript` |
| `pyproject.toml` OR `setup.py` OR `requirements.txt` | `python` |
| `Cargo.toml` | `rust` |
| `*.lua` in root OR `lua/` directory exists | `lua` |
| None detected | `[]` (empty — Serena runs without language server) |

Detection checks markers in the project directory passed to `serena-ensure`. Order doesn't matter — all matches are collected.

## Valuable Serena Tools (Reference)

Tools that the skill should guide Claude to prefer over builtins:

| Serena Tool | Use Instead Of | When |
|-------------|---------------|------|
| `find_symbol` | Grep/Glob | Searching for a function, class, or method by name |
| `find_referencing_symbols` | Grep | Finding all usages of a symbol |
| `get_symbols_overview` | Read (full file) | Understanding file structure without reading contents |
| `replace_symbol_body` | Edit | Replacing an entire function or method body |
| `insert_after_symbol` / `insert_before_symbol` | Edit | Adding code relative to a named symbol |
| `rename_symbol` | Edit `replace_all` | Cross-file renames (imports, qualified names) |
