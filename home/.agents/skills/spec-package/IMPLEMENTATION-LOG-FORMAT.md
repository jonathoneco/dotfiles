# implementation-log.md

Use for packages that will be worked over multiple sessions.

Include:

- completed work
- deviations from plan
- evidence gathered
- remaining cleanup
- worktree or branch state
- handoff notes not already captured in canonical docs

Reference the package instead of duplicating it. Put only session-local state and gotchas here.

## Template

```md
# Implementation Log

## Completed

- ...

## Deviations

- ...

## Evidence

- ...

## Remaining cleanup

- ...

## Handoff notes

- ...
```
