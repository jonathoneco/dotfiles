# 00-design-spec.md

Purpose: canonical intent.

This is the one document every sibling doc defers to. Write it before implementation mechanics.

Include:

- problem and root cause
- current behavior and why it fails
- intended model
- ownership boundaries
- accepted behavior changes
- invariants
- failure modes
- non-goals
- decisions already made
- open decisions
- precedence note: if sibling docs disagree, fix the sibling docs

Do not bury unresolved product or architecture choices in implementation steps.
